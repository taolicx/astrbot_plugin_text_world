# AstrBot 群文游文字世界插件实现规格

## 当前实现定位

这是一个学院都市题材的小时制群文游系统，包含：

- AstrBot 群插件
- SQLite 数据库
- 内置 Web 前端后台
- 每小时统一结算
- 大模型一次输出后由程序拆分群公告和私聊

## 已实现主链路

- 群内 `世界开启` 创建群世界。
- 玩家 `创建角色 游戏名 | 身份设定` 提交角色卡。
- 提交角色卡会创建玩家前端账号，账号为游戏名，初始密码为 QQ 后 6 位。
- 管理员可在前端审核/编辑角色卡，绑定 QQ、身份、阵营、能力、战斗力和状态。
- 玩家审核通过后可每小时提交一次 `行动`。
- 系统按地图连接、状态、NPC、事件、商店和反作弊统一结算。
- 群里发送公开世界公告。
- 玩家私聊收到个人结果。
- 每天配置时间发送玩家状态栏，默认早上 8 点。
- 前端可查看角色、世界、历史、NPC、商店、事件预设。

## 数据库

默认数据库文件：

```text
data/plugin_data/astrbot_plugin_text_world/text_world.sqlite3
```

核心表：

- `users`
- `sessions`
- `worlds`
- `locations`
- `location_edges`
- `characters`
- `npcs`
- `actions`
- `event_presets`
- `inventory`
- `shop_items`
- `relationships`
- `history`

## 权限规则

- 管理员账号可查看和编辑全部数据。
- 玩家账号只能查看自己的角色、所在群公开历史、自己的私密历史、公开商店和基础 NPC 信息。
- 管理员 API 均要求 admin 权限。
- 修改密码会清除旧 session。

## 大模型边界

程序负责权威状态：

- 地图移动
- 生命/精力/水分/饱食度/心情
- 学都币
- 背包
- 商店库存
- 公共事件效果
- 每小时一次行动
- 反作弊

大模型只负责叙事文本：

- 群公告
- 玩家私聊结果

如果大模型失败，程序模板兜底。

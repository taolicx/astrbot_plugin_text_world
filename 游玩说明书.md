# 群文游文字世界游玩说明书

适用插件：`astrbot_plugin_text_world`  
当前版本：`0.3.1`  
世界主题：学院都市小时制文字世界

这是一套运行在 AstrBot 群聊里的文字世界系统。玩家在群里创建角色、每小时提交一次行动，插件统一结算本轮行动，并把公开大事件发到群里，把个人经历私聊发给对应玩家。管理员可以通过 Web 后台审核角色卡、查看世界状态、预设事件、维护商店和查看世界历史。

## 1. 核心玩法概念

### 1.1 这是什么游戏

这是一个“慢节奏群文游”。

玩家不是实时一句一句跑团，而是在每个结算周期内提交一次行动。例如：

```text
行动 我去第七学区中心广场看看今天有没有风纪委员公告。
```

系统会在每小时结算时统一处理所有人的行动。如果两名玩家去了同一个地点，就可能在剧情里相遇。如果玩家与 NPC 出现在同一个地点，也可能触发对话、传闻或事件。

### 1.2 默认结算节奏

- 玩家行动周期：默认每 60 分钟结算一次。
- 公共事件周期：默认每 120 分钟一次。
- 每名玩家每轮只能提交 1 条行动。
- 每天早上 8 点会私聊发送玩家状态栏。
- 管理员可以手动立即结算，也可以手动触发公共事件。

这些周期都可以在 AstrBot 插件配置中调整。

### 1.3 群内与私聊的区别

群内主要显示公开信息：

- 本轮世界公告
- 大事件
- 新闻
- 传闻
- 公开可见的聚集点

私聊主要显示个人信息：

- 你本轮做了什么
- 你遇到了谁
- 你的状态变化
- 你的当前位置
- 本轮提醒

原则是：别人的私密行动结果不会直接发给你。群里只会看到公开世界变化。

## 2. 开始游玩

### 2.1 管理员先开启世界

管理员或群内有权限的人先在目标群发送：

```text
世界开启
```

成功后，插件会在当前群创建一个学院都市世界，并自动种入：

- 23 个学区地图
- 默认路线连接
- 默认 NPC
- 默认商店商品
- 默认公共事件预设
- 世界书背景

如果后台前端里没有地图、NPC、商店或事件，通常是还没有在任何群发送过 `世界开启`。

世界开启后，该群会进入“文游托管模式”：插件会监听群消息，只回复文游指令；普通聊天内容会被静默拦截，不再交给 AstrBot 正常聊天模型输出。这样可以避免群内闲聊把世界叙事打乱。

### 2.2 玩家查看帮助

发送：

```text
世界帮助
```

也可以使用别名：

```text
文游帮助
世界菜单
```

### 2.3 玩家查看角色卡格式

发送：

```text
角色卡模板
```

也可以使用：

```text
角色模板
创建格式
```

系统会返回完整角色卡格式。

## 3. 玩家命令大全

### 3.1 创建角色

格式：

```text
创建角色 游戏名 | 身份 | 阵营 | 能力 | 能力等级
```

示例：

```text
创建角色 星野遥 | 第七学区高中生，风纪委员见习 | 风纪委员 | 微弱电磁感应，可察觉近距离异常电流，连续使用会头痛 | Level 2
```

也可以用自然语言提交：

```text
创建角色 我叫星野遥，是第七学区高中生兼风纪委员见习，能力是微弱电磁感应，Level 2。
```

自然语言角色卡会优先交给配置里的快速模型整理成标准字段；如果 API 不可用，插件会使用程序规则兜底整理。整理后仍然进入待审核状态，不会绕过管理员审核。

可用别名：

```text
角色创建
创建人物
```

提交后角色卡会进入待审核状态。管理员审核通过前，玩家不能提交行动、签到或正式参与结算。

#### 字段说明

游戏名：
角色在世界里的名字。前端玩家账号也默认使用这个名字。

身份：
角色的社会身份，例如学生、风纪委员见习、研究所实习生、留学生、Skill-Out 边缘成员、便利店兼职学生等。

阵营：
角色倾向或所属组织，例如普通学生、风纪委员、警备员协助者、研究机构、暗部边缘、留学生、无阵营等。

能力：
角色的能力设定，要写清楚表现、限制和代价。建议能力有边界，不要写成全知全能。

能力等级：
建议使用学院都市体系：

```text
Level 0
Level 1
Level 2
Level 3
Level 4
```

`Level 5` 建议只允许管理员特批。`Level 6`、绝对能力者、无限能力、全图瞬移等内容会被视为高风险设定或作弊倾向。

### 3.2 提交行动

格式：

```text
行动 你这一轮要做什么
```

示例：

```text
行动 我从第七学区校门口前往第七学区主干道，顺路观察警备机器人巡逻路线。
```

可用别名：

```text
文游行动
提交行动
```

行动提交后，本轮不能再提交第二条。系统会提示当前已有多少人提交行动。

### 3.3 查看状态

发送：

```text
状态
```

可用别名：

```text
我的状态
状态栏
```

状态栏会显示：

- 身份
- 阵营
- 能力
- 战斗力/能力等级
- 审核状态
- 当前位置
- 生命
- 精力
- 水分
- 饱食度
- 心情
- 学都币
- 背包
- 本轮行动是否已提交
- 下次结算时间

### 3.4 查看地图

发送：

```text
地图
```

可用别名：

```text
世界地图
```

如果你已经有角色，地图会优先显示：

- 当前位置
- 当前位置描述
- 当前可前往地点
- 全部地点连接

### 3.5 查看本轮待结算情况

发送：

```text
待结算
```

可用别名：

```text
本轮行动
行动列表
```

系统会显示：

- 当前轮次
- 已提交行动人数
- 已审核角色数
- 下次结算时间

### 3.6 每日签到

发送：

```text
签到
```

可用别名：

```text
每日签到
```

签到需要角色卡已审核通过。默认每日奖励一定数量学都币，具体数值可在插件配置里调整。

### 3.7 绑定私聊

玩家需要先私聊 bot，然后发送：

```text
绑定文游私聊
```

可用别名：

```text
绑定世界私聊
文游私聊绑定
```

绑定后，系统才能稳定私聊发送：

- 每轮个人结果
- 每天早上 8 点状态栏

如果没有绑定私聊，插件会尝试 OneBot 私聊兜底，但具体能否成功取决于平台和 bot 权限。

模型路由：
查看主模型、剧情模型、快速模型和默认模型的 provider 配置。

账号：
修改当前后台账号密码。

## 6. 世界地图规则

### 6.1 地图不是装饰

地图是硬规则。玩家不能随便从任意地点瞬移到任意地点。

如果玩家行动中写了目标地点，插件会检查当前位置和目标地点是否相邻。

如果路线成立：

```text
你从 A 地点前往 B 地点。
```

如果路线不成立：

```text
你尝试前往某地，但地图路线不成立，本轮留在原地。
```

### 6.2 默认地图范围

默认包含学院都市 23 个学区，以及一些常用细分地点：

- 第七学区校门口
- 第七学区主干道
- 第七学区某高中
- 书库终端图书馆
- 能力开发实验楼
- 第七学区学生宿舍
- 学生食堂
- 第十五学区商业街
- 第七学区中心广场
- 冥土追魂医院
- 第一至第二十三学区

### 6.3 如何规划行动

建议玩家先发送：

```text
地图
```

看清楚“可前往”地点后，再提交行动。

好的行动示例：

```text
行动 我从第七学区校门口去第七学区主干道，查看公告栏附近有没有新的传闻。
```

不建议的行动：

```text
行动 我瞬移到第二十三学区机场。
```

这种行动会被反作弊或地图规则拦截。

## 7. 状态、货币与背包

### 7.1 状态项

生命：
代表受伤程度。生命过低时，本轮只能休息或去医院治疗。

精力：
代表体力和行动状态。移动、工作、冒险都会消耗精力。

水分：
每小时结算会下降。水分过低会影响精力和生命。

饱食度：
每小时结算会下降。饱食度过低会影响心情和精力。

心情：
代表精神状态，会受事件、休息、食物、风险行动影响。

学都币：
世界内货币，可通过签到、打工、事件奖励等方式获得，也会用于购买物品和治疗。

### 7.2 每小时自然消耗

每次结算后，玩家会自然消耗一定水分和饱食度。  
如果长期不补给，角色状态会逐渐变差。

### 7.3 商店与购买

行动里出现购买意图时，系统会尝试匹配商店物品。

示例：

```text
行动 我去学生食堂买一份学生便当，再找空位休息一下。
```

购买成功后：

- 扣除学都币
- 物品进入背包
- 根据商品效果恢复水分、饱食度、生命、精力或心情

默认商品包括：

- 矿泉水
- 营养果冻
- 学生便当
- 校园午餐券
- 能量饮料
- 便携绷带
- 医疗凝胶贴
- 通勤一日券
- 书库资料卡
- 便携终端电池
- 研究区临时通行证
- 防水学生包

### 7.4 医院治疗

如果角色受伤，可以提交类似行动：

```text
行动 我去冥土追魂医院处理伤口，接受基础治疗。
```

治疗通常会消耗学都币，并恢复生命。

### 7.5 打工赚钱

如果行动包含兼职、打工、帮忙、跑腿、工作等内容，系统可能判定为打工行动。

示例：

```text
行动 我去第十六学区找一份短时跑腿兼职，赚点学都币。
```

打工通常会获得学都币，但会消耗精力。

## 8. NPC 与好感度

### 8.1 NPC 会移动

NPC 每轮有概率随机移动到相邻地点。  
如果玩家和 NPC 出现在同一地点，就可能在个人结果里遇到该 NPC。

### 8.2 默认 NPC 类型

默认 NPC 包含：

- 风纪委员支部情报支援
- 风纪委员空间移动能力者
- 警备员兼教师
- 第七学区医院医生
- 某高中教师
- 统括理事会书记官
- 暗部边缘联络员
- 第十一学区货检员
- 第十五学区频道编辑
- 第三学区接待员
- 第二十学区运动工学教练
- 第二十一学区水源区观测员

### 8.3 好感度基础波动

当玩家与玩家或玩家与 NPC 相遇时，系统有概率产生小幅好感度变化。

这个波动很小，不会因为一次见面就大幅改变关系。长期互动才会逐渐形成稳定关系。

## 9. 公共事件与世界历史

### 9.1 自动公共事件

默认每 2 小时可能出现公共事件。公共事件会影响世界，也可能对玩家状态造成轻微影响。

默认事件包括：

- 大霸星祭筹备公告
- 第四学区限时餐饮折扣
- 第十一学区物流安检升级
- 第十五学区都市传说专题
- 第十八学区能力测定日
- 第二十一学区水库调度
- 第二十二学区地下街停电演习
- 第十学区少年院异动传闻
- 警备机器人巡逻升级
- 书库终端维护

### 9.2 管理员预设事件

管理员可以在后台“事件”页签新增事件。

事件可以只保存为预设，也可以勾选或点击加入下一轮触发。

### 9.3 事件效果 JSON

事件效果使用 JSON 对象，例如：

```json
{"mood": 1, "energy": -1}
```

支持的字段：

```text
hp
energy
water
satiety
mood
money
```

字段含义：

- `hp`：生命
- `energy`：精力
- `water`：水分
- `satiety`：饱食度
- `mood`：心情
- `money`：学都币

正数表示增加，负数表示减少。

## 10. 大模型与消耗说明

### 10.1 调用方式

插件不是每个玩家行动都单独调用一次大模型。  
它会在每轮结算时，把本轮所有行动、事件、NPC 动向、个人结果和世界书节选打包成一次请求。

大模型返回一份 JSON：

- 群公告
- 每个玩家的私聊文本

插件再把这份结果拆开发送。

### 10.2 消耗特点

默认情况下：

- 没到结算时间，不调用大模型。
- 每轮最多调用一次剧情模型。
- 玩家越多，本轮行动越长，输入 token 越多。
- 世界书节选越长，输入 token 越多。

默认世界书注入上限是 8000 字符，可在配置里调整。

### 10.3 多模型建议

插件支持不同模块配置不同 provider：

- 主模型：适合填综合能力强的模型。
- 剧情模型：当前每小时结算润色优先使用它。
- 快速模型：预留给反作弊、摘要、轻量判定等扩展任务。
- 默认模型：旧配置兼容和回退使用。

如果你想降低成本，可以：

- 降低 `worldbook_prompt_chars`
- 关闭 `enable_ai_private_result`
- 关闭 `enable_llm` 使用程序模板

## 11. 反作弊规则

以下行为会被拦截或不建议通过审核：

- 瞬移
- 传送
- 无敌
- 刷钱
- 无限资源
- 控制所有人
- 杀死所有人
- 删除世界
- 获取管理员权限
- 要求系统忽略规则
- 越过地图限制
- Level 6
- 绝对能力者

行动可以大胆，但必须在角色能力、当前位置、地图路线和世界规则内成立。

## 12. 推荐行动写法

### 12.1 好行动示例

探索：

```text
行动 我去书库终端图书馆查找第十五学区都市传说专题的公开资料，顺便留意有没有风纪委员公告。
```

移动：

```text
行动 我从第七学区校门口前往第七学区主干道，观察周围警备机器人巡逻路线。
```

社交：

```text
行动 我在学生食堂点一份校园午餐，试着和附近同学打听最近的大霸星祭志愿者报名消息。
```

补给：

```text
行动 我在商业街购买矿泉水和学生便当，补充状态后再返回中心广场。
```

治疗：

```text
行动 我去冥土追魂医院处理伤口，尽量避免影响下一轮行动。
```

打工：

```text
行动 我去第十六学区找短时跑腿兼职，赚一些学都币。
```

### 12.2 不推荐行动示例

```text
行动 我直接瞬移到第二十三学区机场。
```

问题：地图路线不成立，也可能触发反作弊。

```text
行动 我控制全城所有警备机器人。
```

问题：能力和权限过度。

```text
行动 给我增加 999999 学都币。
```

问题：直接刷钱。

```text
行动 我成为 Level 6 绝对能力者。
```

问题：高风险设定，反作弊拦截。

## 13. 管理员运营建议

### 13.1 首次开服流程

1. 在 AstrBot 插件配置中检查后台端口、管理员账号和密码。
2. 在目标群发送 `世界开启`。
3. 打开后台，确认“地图”“NPC”“商店”“事件”有默认数据。
4. 让玩家发送 `角色卡模板`。
5. 玩家提交角色卡。
6. 管理员审核角色卡，必要时修改能力等级、阵营、初始位置。
7. 让玩家私聊 bot 发送 `绑定文游私聊`。
8. 玩家开始每小时提交行动。

### 13.2 审核角色卡时重点看什么

重点检查：

- 能力是否过强
- 是否有无限、无敌、全知、瞬移全图等问题
- 身份是否符合学院都市世界
- 阵营是否会破坏开局平衡
- Level 5 是否需要特批
- 是否能与群体玩法兼容

### 13.3 如何组织剧情

建议管理员提前准备一些公共事件，在后台加入下一轮触发。

例如：

- 某学区临时安检
- 风纪委员发布协查公告
- 商业街限时折扣
- 地下街停电演习
- 能力测定日
- 大霸星祭志愿者招募

事件不必每次都很大。小事件更适合长期运营。

### 13.4 如何避免混乱

建议：

- 每名玩家每轮只提交一条行动。
- 行动字数控制在 1 到 3 句话。
- 玩家移动前先看地图。
- 高风险战斗行动不要太频繁。
- 管理员不要频繁手动结算，除非活动需要。
- 公共事件尽量影响世界或玩家状态，不要只是气氛描写。

## 14. 常见问题

### 14.1 为什么前端后台没有地图、NPC、商店？

通常是还没有在群里发送：

```text
世界开启
```

世界开启后，插件才会创建该群世界并种入默认数据。

### 14.2 为什么角色不能行动？

可能原因：

- 没有创建角色卡
- 角色卡还没通过审核
- 本轮已经提交过行动
- 生命值过低，只能休息或治疗
- 行动包含反作弊高风险词

### 14.3 为什么没有收到私聊结果？

可能原因：

- 没有私聊 bot 发送 `绑定文游私聊`
- 平台不允许 bot 主动私聊
- bot 没有私聊权限
- 私聊 fallback 失败

建议玩家先主动私聊 bot 并绑定。

### 14.4 为什么行动没有按我想的发生？

系统会优先遵守硬规则：

- 地图路线
- 角色状态
- 货币
- 背包
- 审核状态
- 反作弊

如果行动写得太夸张，系统会降级处理或拦截。

### 14.5 大模型会不会乱改状态？

不会。状态、位置、货币、背包等由程序结算。  
大模型只负责把程序结果润色成更像剧情的文本。

### 14.6 能不能不用大模型？

可以。关闭 `enable_llm` 后，插件会使用程序模板输出结果。

### 14.7 可以多群联动吗？

当前每个群是独立世界。多群联动属于后续扩展方向。

### 14.8 能不能自定义地图？

当前后台可查看地图，默认地图由世界书种入。更完整的地图编辑器属于后续扩展方向。管理员如果需要深度自定义，建议后续扩展专门的地图编辑页。

03每小时行动怎么写

地点

写清楚你从哪里去哪里。系统会按地图路线判断，不能突然瞬移。

目标

写清楚你想调查、购买、休息、寻找谁，AI 才好给结果。

互动

你和其他玩家或 NPC 去到同一地点，就可能触发相遇和剧情。

行动示例

行动 我去便利店购买饮用水和便当，然后返回宿舍休息。
行动 我留在当前位置，和附近的 NPC 打听最近的异常事件。

更好的行动

行动 我尝试和星野遥同行，前往第七学区医院附近调查受伤学生的传闻。
不建议：行动 我去全地图所有地方调查一遍并获得全部真相。

05系统规则和后台

运行节奏

每小时结算一次行动；每两小时触发公共事件；每天早上 8 点私聊状态栏。

个人状态

生命、精力、水分、饱食度、心情、学都币、背包、好感度、阵营都会影响玩法。

后台页签

总览、角色、事件、商店、地图、NPC、历史、世界书、模型路由。

管理员做什么

审核 / 编辑角色卡，设置位置和状态，添加预设事件，把事件加入下一轮，维护商店商品和库存，查看世界历史。

私聊和群聊

群里看公共世界变化；私聊收个人结果、状态栏、地图、本轮和签到。别人的私聊结果不会直接发给你。
06世界书、地图和模型路由

地图硬约束

移动要符合地点连接；目标不在可达路线中时，本轮会被拦回原地。

模型路由

最短开玩路线：世界开启 → 角色卡模板 → 创建角色 → 私聊 绑定文游私聊 → 地图 → 行动。

系统怎么跑
后台怎么看

这页讲的是“系统本身怎么运转”：每小时结算、每两小时公共事件、每天早上 8 点私聊状态栏、后台页签、模型路由、世界书和地图约束。

结算节奏

每小时结算一次行动。

每两小时一次公共事件。

每天早上 8 点发私聊状态栏。

角色与资源

生命、精力、水分、饱食度、心情、学都币、背包、好感度、阵营一起跑。

硬规则

角色卡先审核；每人每小时 1 条行动；地图按路线走，不能瞬移；受伤后可以去医院。

后台页签

总览世界数量、角色数量、待审核、下轮事件、商品和历史 KPI。
角色审核 / 编辑角色卡，玩家看自己的状态。
事件添加预设事件，并可加入下一轮结算。
商店添加商品、库存和效果 JSON。
NPC查看当前身份、阵营、位置和态度。
历史查看公共 / 私聊历史记录。
世界书查看是否接入、文件大小、注入字符数和预览。
模型路由主 / 剧情 / 快速 provider 的配置和回退结果。

模型路由

主模型：Claude 4.6o，负责默认高质量任务和复杂判断。
剧情模型：DeepSeek v4 Pro，优先负责每小时结算、公共公告和私聊润色。
快速模型：DeepSeek v4 Flash，处理角色卡整理、摘要、反作弊和轻量判断等低延迟任务。
回退顺序：剧情 = story → main → 群默认 provider → default；快速 = fast → main → 群默认 provider → default。
如果你想省消耗，可以关 `enable_llm`，或者只关 `enable_ai_private_result`，让私聊结果固定走程序模板。

地图与移动

地图负责硬约束：玩家不能瞬移，移动要符合地点连接；如果目标不在路线里，本轮会被拦回原地。

世界书与反作弊

世界书默认是 `worldbook/academy_city_worldbook.txt`，通过 `worldbook_prompt_chars` 节选注入，默认不整本塞给模型，避免 token 爆炸。
反作弊会拦高风险设定和异常收益，能力等级默认不建议超过 Level 5。
世界开启后，群内普通聊天静默，只回复文游指令；个人结果和早八状态优先私聊发送。
如果需要更省 token，可以把 `worldbook_prompt_chars` 调到 4000 - 8000 之间测试。
群里看公共世界变化，私聊看个人结果。别人的私聊结果不会直接回到你这里。
这套系统是“程序硬规则 + 大模型润色”混合跑法，不是纯聊天机器人。

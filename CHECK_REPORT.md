## 十轮结论

1. 角色保存与账号边界：修复后台编辑已有角色时把生命/水分/饱食度等重置为默认值的问题；新增审核状态和地点 key 校验。
2. 结算状态与事件：修复预设事件使用一次后无法再次点击触发的问题。
3. 商店与背包：新增商品效果 JSON 对象校验、库存下限保护、购买时脏效果兜底；后台显式填写密码时会真正重置玩家密码。
4. 前端权限与数据范围：修复玩家输入内容直接 `innerHTML` 渲染的 XSS 风险；新增玩家可见的“我的角色”和“商店”区域。
5. 定时任务与发送：结算返回群公告和私聊发送成功率，私聊失败不会中断整轮。
6. 大模型输出：`enable_ai_private_result` 开关现在生效；JSON 解析支持代码块包裹并保留模板兜底。
7. 数据库迁移：补充缺列迁移和高频查询索引，降低旧库升级失败和查询变慢风险。
8. 配置与文档：同步新增开关、玩家账号/重置密码说明和大模型消耗说明。
9. 端到端回归：核心业务链路通过临时库模拟。
10. 最终清理：编译通过，待清理 `__pycache__` 后交付。

## 再次十轮自查结论

### 为什么多轮检查仍会发现问题

这个插件本质上是一个小型文字游戏服务器，而不是单个命令脚本。它同时包含账号系统、Web 前端、SQLite 迁移、群命令、定时结算、大模型润色、商店背包和事件效果。前几轮主要查功能正确性，后续切到“安全”“旧库升级”“运营误操作”“坏输入”“并发结算”等视角时，会照出不同类别的问题。单靠人工阅读很难一次覆盖所有交叉组合，因此这次补了更多可重复模拟。

### 本轮新增修复

- 配置布尔值支持字符串 `false`、`0`、`no`、`off` 等写法，避免配置界面传字符串时被 Python `bool("false")` 误判为启用。
- 管理员重置玩家密码时要求至少 6 位，并清理该玩家旧 session。
- Web API 限制请求体大小，拒绝非对象 JSON，事件触发 id 做明确校验。
- 空 QQ 号不能创建角色、绑定私聊或提交行动。
- 生命值为 0 的角色只能休息或治疗，不能继续普通行动。
- 手动结算和定时结算增加同群世界锁，避免重复推进同一轮。
- 事件效果必须是 JSON 对象。
- 旧库迁移时给历史角色回填玩家前端账号。
- 大模型返回文本增加长度裁剪，避免异常长输出刷屏。
- 治疗/休息/购买识别补充英文关键词和商品名触发。

### 本轮新增验证

- 配置字符串布尔值解析：通过。
- 低生命行动限制和治疗例外：通过。
- 重置密码后旧 session 失效：通过。
- 大模型超长输出裁剪：通过。
- 旧库角色账号回填：通过。
- Python 源码编译：通过。
- 前端脚本解析：通过。

## 不变量逻辑复查结论

这轮改用“不变量”而不是“模块”来查：身份不能串、轮次不能重复、资源不能异常、玩家不能看后台隐藏数据、旧库升级不能破坏账号、大模型不能新增私聊对象。

### 本轮新增修复

- 未审核角色不能签到拿学都币。
- 玩家前端商品数据不再返回 `effect_json`，避免泄露后台隐藏效果。
- 触发不存在的预设事件会明确报错，不再静默成功。
- 旧库迁移回填账号时，跨群同名不同 QQ 不会串到同一玩家账号；和管理员账号重名的旧角色会自动追加 QQ 后缀。
- 旧角色账号回填顺序调整为管理员账号创建之后，避免抢占管理员账号名。

### 本轮新增验证

- 轮次推进不变量：连续结算只按轮次递增，不重复处理已结算行动。
- 玩家数据隔离：玩家快照只能看到自己角色、公开历史和自己的私密历史。
- 资源边界：坏价格/库存输入不会导致负库存或崩溃。
- 旧库迁移：同名不同 QQ、角色名撞管理员账号均可安全回填。
- 大模型污染：模型返回额外 QQ key 时不会新增私聊对象。
- Python 源码编译和前端脚本解析均通过。

## 私聊与群聊上下文排查

### 检查范围

- 群聊创建角色、群聊签到、私聊签到、私聊绑定。
- 数据库 `worlds`、`characters`、`history` 的实际记录。
- AstrBot 消息事件中的群聊/私聊类型与 `group_id`、`session_id` 差异。

### 复现现象

- 玩家在群 `710187710` 创建并审核通过角色卡。
- 群聊 `签到` 可正常获得学都币，并写入私密历史。
- 同一 QQ 在私聊发送 `签到` 时，插件提示未创建角色。

### 数据证据

- `characters` 表中存在 `group_id=710187710`、`qq_id=1556592332` 的角色，状态为 `approved`。
- 该角色已经绑定 `private_origin=napcat:FriendMessage:1556592332`。
- `history` 表已有该角色的群世界签到记录。

### 根因

私聊命令也会命中通用 `@filter.command("签到")` 处理器。旧版 `_group_id()` 在拿不到真实群号时，会继续读取 `message_obj.session_id`；但私聊事件的 `session_id` 是用户会话 ID，不是群号。于是私聊 `签到` 实际查询的是 `group_id=1556592332` 一类的私聊会话世界，而角色真实保存在 `group_id=710187710` 下，所以提示未创建角色。

### 修复方向

- `_group_id()` 不再把私聊 `session_id` 当作群号。
- 私聊 `状态`、`地图`、`待结算`、`签到` 通过 QQ 号和 `private_origin` 反查玩家绑定的群世界。
- `创建角色`、`行动`、`世界开启` 仍要求在群聊中使用，避免在私聊误创建独立世界。

# 群文游文字世界 AstrBot 插件

这是一个学院都市题材的小时制群文游系统，包含 AstrBot 群插件、SQLite 数据库和内置前端后台。

## 已实现的 v1/MVP 功能

- 群内开启世界
- 玩家提交角色卡，支持标准格式和自然语言描述
- 角色卡完整格式：身份、阵营、能力、能力等级
- 世界开启后静默拦截群内普通聊天，避免 AstrBot 正常聊天模型抢答；只回复文游指令
- 管理员前端审核/编辑角色卡并绑定 QQ 号
- 每小时提交一次行动
- 每小时统一结算
- 大模型一次性输出群公告和所有玩家私聊结果，程序拆分发送
- 大模型失败时使用程序模板兜底
- 每两小时公共事件
- 管理员前端预设事件，并加入下一轮结算
- 地图移动约束，防止瞬移
- 按世界书自动种入 23 学区地图、默认 NPC、商店和事件预设
- NPC 随机行动
- 玩家/NPC/玩家相遇判断
- 学都币、水分、饱食度、生命、精力、心情
- 背包和基础商店框架
- 管理员前端添加商品
- 玩家前端查看自己的角色、状态、可见历史和商店
- 学院都市世界书随插件打包，并按配置节选注入大模型
- 增强版 WebUI：总览 KPI、筛选搜索、角色/事件/商店/NPC/历史/世界书页签
- 每日签到加学都币
- 私聊绑定后可在私聊提交行动、查看状态、地图、待结算和签到，数据仍归属原群世界
- 每天早上 8 点私聊状态栏
- 玩家和 NPC、玩家和玩家之间的好感度基础波动
- 世界历史查询

## 前端后台

默认地址：

```text
http://127.0.0.1:8787
```

默认管理员账号：

```text
admin
```

默认管理员密码：

```text
admin123456
```

上线前请在 AstrBot 插件配置里修改管理员密码。

玩家前端账号会在提交角色卡或管理员创建角色时生成：账号为游戏名，初始密码默认是 QQ 后 6 位。管理员在角色表单里填写“初始/重置密码”时，会重置该玩家账号密码。

新版后台包含：

- 总览：世界数量、角色数量、待审核、下轮事件、商品和历史 KPI。
- 角色：管理员审核/编辑角色卡，玩家查看自己的状态。
- 事件：管理员添加预设事件，并可加入下一轮结算。
- 商店：管理员添加商品、库存和效果 JSON。
- NPC：查看 NPC 当前身份、阵营、位置和态度。
- 历史：查看可见的公共/私聊历史记录。
- 世界书：查看世界书是否接入、文件大小、注入字符数和预览。
- 模型路由：查看主模型、剧情模型、快速模型的 provider 配置和回退结果。

## 世界书

插件内置世界书文件：

```text
worldbook/academy_city_worldbook.txt
```

默认启用 `worldbook_enabled`，并通过 `worldbook_prompt_chars` 控制每次结算传给大模型的世界书节选长度，默认 8000 字符。完整世界书会随插件打包，WebUI 可查看状态和预览，但不会在每轮结算时整本注入，避免 token 消耗失控。

```text
data/plugin_data/astrbot_plugin_text_world/text_world.sqlite3
```

## 推荐使用流程

1. 群里发送 `世界开启`。
2. 玩家先发送 `角色卡模板`，再按 `创建角色 游戏名 | 身份 | 阵营 | 能力 | 能力等级` 提交；也可以在 `创建角色` 后直接写自然语言描述，插件会优先用快速模型整理成标准角色卡。
3. 管理员打开前端后台审核角色卡，补充阵营、能力、战斗力与位置。
4. 玩家私聊 bot 发送 `绑定文游私聊`。
5. 玩家每小时在群里发送 `行动 内容`。
6. 系统定时结算并私聊个人结果，群里发送公共公告。

## 设计边界

程序负责硬规则：地图、状态、货币、背包、每小时一次行动、反作弊和数据库。

大模型负责叙事：群公告和个人结果文本润色。模型不能直接决定最终状态，避免 AI 随意改世界。配置里可以关闭 `enable_llm` 使用纯程序模板，也可以只关闭 `enable_ai_private_result`，让私聊结果固定使用程序权威模板、仅群公告走大模型。

## 世界书落地内容

当前插件已按世界书默认生成：

- 23 个学区地图和路线连接
- 默认 NPC：第七学区日常角色、常盘台角色、风纪委员、警备员、医院、研究侧、暗部边缘、Level 5 传闻与幕后禁区角色
- 默认商店：水、便当、医疗、通勤、终端电池等日常物资
- 默认事件：大霸星祭、物流安检、地下街停电、能力测定日等公共事件
- 角色卡模板：身份、阵营、能力、能力等级

## 多模型路由建议

先在 AstrBot 里把不同 API / 模型配置成不同 provider，再在插件配置里选择：

- `main_provider_id`：主模型，适合填 Claude 4.6o 一类综合能力强的模型。
- `story_provider_id`：剧情模型，当前每小时结算润色会优先走这里，适合填 DeepSeek v4 Pro。
- `fast_provider_id`：快速模型，用于自然语言角色卡整理，并预留给后续反作弊、摘要、轻量判定等任务，适合填 v4 Flash。
- `default_provider_id`：旧版兼容默认 provider，未配置专用模型时作为回退。

当前实际会产生大模型调用的是“每轮结算剧情润色”和“自然语言角色卡整理”。结算润色调用顺序是：`story_provider_id` -> `main_provider_id` -> 当前群会话 provider -> `default_provider_id`。自然语言角色卡整理调用顺序是：`fast_provider_id` -> `main_provider_id` -> 当前群会话 provider -> `default_provider_id`；API 失败会自动使用程序解析兜底。

## 大模型消耗说明

每轮结算最多调用一次大模型：程序把本轮行动、公共事件、NPC 动向、个人结果和可控长度的世界书节选打包成一个 JSON 请求，模型一次性返回群公告和各玩家私聊文本，插件再拆开发送。没有到结算周期不会持续调用；关闭大模型后不产生模型调用。

世界书默认节选 8000 字符，通常会额外增加几千 token 的输入消耗。若玩家多、每轮行动多，建议先把 `worldbook_prompt_chars` 调到 4000-8000 之间观察效果；如果只想使用程序模板，可关闭 `enable_llm`。

结算输出长度可在配置里调整：`public_summary_max_chars` 控制群公告最大字符数，`private_result_max_chars` 控制每个玩家私聊结算最大字符数。调低可以减少输出消耗，调高可以让剧情更完整；默认分别是 1800 和 1400，当前上限均为 12000 字符。

长消息会按 `message_chunk_chars` 自动分段发送，默认每段 1800 字符，段间隔由 `message_chunk_delay_ms` 控制。

冲突不再按“过家家训练”处理：明确攻击、互殴、袭击、正面冲突会触发 PvP 伤害，默认生命损失区间由 `pvp_damage_min` / `pvp_damage_max` 控制；高风险行动额外受 `risk_damage_multiplier` 影响。

如果玩家行动影响到其他玩家，例如殴打、袭击、强行互动等，即使目标玩家本轮没有提交行动，系统也会生成一条被动影响结果并尝试私聊发送给目标玩家，同时写入目标玩家状态变化。

玩家交互会按类型处理：同行/组队、交流、支援保护、压力型互动、赠与/交易、冲突。系统会为双方补充私聊反馈，小幅调整玩家之间的关系分；涉及明确金额或背包物品名称时，会尝试完成学都币或物品转移。

资源转移采用“支出方确认”规则：只有行动者写明自己给出/转账/赠送自己的学都币或背包物品，才会过账；单方面声明“某玩家给我钱/物品”会被反作弊拦截；“抢夺/偷取/勒索”会按冲突或压力行动结算，但不会直接把对方资源转到自己名下。

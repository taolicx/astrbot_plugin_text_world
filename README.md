# 群文游文字世界 AstrBot 插件

这是一个学院都市题材的小时制群文游系统，包含 AstrBot 群插件、SQLite 数据库和内置前端后台。

## 已实现的 v1/MVP 功能

- 群内开启世界
- 玩家提交角色卡，支持标准格式和自然语言描述
- 角色卡完整格式：身份、阵营、能力、能力等级、性别、穿衣着装、简易身材
- 世界开启后静默拦截群内普通聊天，避免 AstrBot 正常聊天模型抢答；只回复文游指令
- 管理员前端审核/编辑角色卡并绑定 QQ 号
- 管理员可在群聊内直接查看待审核角色并批准、拒绝或退回角色卡
- 每小时提交一次行动
- 每小时统一结算
- 大模型一次性输出公共摘要和所有玩家私聊结果，程序默认只私聊已审核玩家，不再往群里发送剧情公告
- 大模型失败时使用程序模板兜底
- 每两小时公共事件
- 管理员前端预设事件，并加入下一轮结算
- 地图移动约束，防止瞬移
- 按世界书自动种入 23 学区地图、默认 NPC、商店和事件预设
- NPC 随机行动
- 玩家/NPC/玩家相遇判断
- 学都币、水分、饱食度、生命、精力、心情
- 全局物价倍率，可在配置中调高或调低商店实际结算价
- 休息会大幅恢复精力，设施休息或付费休息恢复更多
- 濒死不再扣除能力经验
- 背包和基础商店框架
- 管理员前端添加商品
- 玩家前端查看自己的角色、状态、可见历史和商店
- 学院都市世界书随插件打包，并按配置节选注入大模型
- 增强版 WebUI：总览 KPI、筛选搜索、角色/事件/商店/NPC/历史/世界书页签
- 每日签到加学都币
- 私聊绑定后可在私聊提交行动、查看状态、地图、待结算和签到，数据仍归属原群世界
- 每天早上 8 点私聊状态栏
- 玩家和 NPC、玩家和玩家之间的好感度基础波动，数值隐藏，只通过剧情态度体现
- `任务/今日任务/引导` 指令，可按身份、现实时间、当前位置生成可选任务和成长方向
- 身份权限识别：常盘台/学舍之园、栅川、某高中、风纪委员、警备员、研究机构等会按身份和阵营共同判定
- NPC 会按现实时间和自身身份修正活动范围，避免佐天泪子夜间出现在常盘台这类不合理地点
- 玩家移动途经多段路线时，路上可能触发传闻、熟人擦肩或 NPC 介入
- 扩展经验：体力、社交、学业、艺术会影响战斗抗压、互动消耗、课程收益和部分打工收益
- 世界历史查询
- 群聊默认只用于注册、审核、查询和玩家指令，不再发布结算剧情公告；如需恢复群公告可打开 `send_public_summary_to_group`

## 前端后台

默认地址：

```text
http://127.0.0.1:8787
```

默认管理员账号：

```text
admin
```

默认管理员密码：

```text
admin123456
```

上线前请在 AstrBot 插件配置里修改管理员密码。

玩家前端账号会在提交角色卡或管理员创建角色时生成：账号为游戏名，初始密码默认是 QQ 后 6 位。管理员在角色表单里填写“初始/重置密码”时，会重置该玩家账号密码。

新版后台包含：

- 总览：世界数量、角色数量、待审核、下轮事件、商品和历史 KPI。
- 角色：管理员审核/编辑角色卡，玩家查看自己的状态。
- 事件：管理员添加预设事件，并可加入下一轮结算。
- 商店：管理员添加商品、库存和效果 JSON。
- NPC：查看 NPC 当前身份、阵营、位置和态度。
- 历史：查看可见的公共/私聊历史记录。
- 世界书：查看世界书是否接入、文件大小、注入字符数和预览。
- 模型路由：查看主模型、剧情模型、快速模型的 provider 配置和回退结果。

## 世界书

插件内置世界书文件：

```text
worldbook/academy_city_worldbook.txt
```

默认启用 `worldbook_enabled`，并通过 `worldbook_prompt_chars` 控制每次结算传给大模型的世界书节选长度，默认 8000 字符。完整世界书会随插件打包，WebUI 可查看状态和预览，但不会在每轮结算时整本注入，避免 token 消耗失控。

## 推荐使用流程

1. 群里发送 `世界开启`。
2. 玩家先发送 `角色卡模板`，再按 `创建角色 游戏名 | 身份 | 阵营 | 能力 | 能力等级 [| 性别 | 穿衣着装 | 简易身材]` 提交；也可以在 `创建角色` 后直接写自然语言描述，插件会优先用快速模型整理成标准角色卡。
3. 管理员打开前端后台审核角色卡，或在群聊使用 `待审核角色` 和 `审核角色 游戏名或QQ 通过` 快速审核；需要退回时使用 `审核角色 游戏名或QQ 拒绝/退回`。
4. 玩家私聊 bot 发送 `绑定文游私聊`。
5. 玩家每小时在群里发送 `行动 内容`。
6. 系统定时结算，将公共事件摘要和个人结果私聊给已审核玩家；群里默认不发送剧情公告。

## 设计边界

程序负责硬规则：地图、状态、货币、背包、每小时一次行动、反作弊和数据库。

大模型负责叙事：公共摘要和个人结果文本润色。模型不能直接决定最终状态，避免 AI 随意改世界。配置里可以关闭 `enable_llm` 使用纯程序模板，也可以只关闭 `enable_ai_private_result`，让个人结果固定使用程序权威模板、仅公共摘要走大模型。

如果大模型返回空文本、JSON 异常、缺少某个玩家结果，或输出“你的行动被记录并执行，具体细节会在本轮剧情里展开”这类占位句，插件会自动回退到程序已经结算好的公共摘要/个人结果，避免结算后无内容可发。

世界时间与现实北京时间同步。状态栏、待结算、公共摘要、私聊结果和大模型润色输入都会带当前北京时间、星期和时段；剧情里的昼夜、上课/放学、店铺营业、人流和公共交通应按现实时间处理。

周期同步规则：`cycle_minutes` 和 `event_cycle_minutes` 会写入每个群世界的数据库记录。插件启动、重载、调度器轮询、玩家/管理员指令入口和内置 WebUI 快照请求都会检测最新配置；发现周期变化时会同步到已有世界，并从当前时间重新计算下次结算和下次事件时间，避免后台配置改了但旧世界仍沿用旧周期。

事件发送规则：`include_public_summary_in_private` 默认开启，公共事件、大事件、新闻传闻会合并进所有已审核玩家的私聊；本轮没行动的玩家也会收到事件摘要。`send_public_summary_to_group` 默认关闭，因此群聊只保留注册、审核、地图、状态、待结算等指令回复，不发布剧情公告。`ambient_event_chance_percent` 控制未行动玩家和环境反馈的小事件概率，调高后更容易出现隔壁宿舍、窗外、走廊、广播或附近 NPC 的动静。

经济与恢复规则：`shop_price_multiplier_percent` 控制全局物价倍率，100 为默认，50 半价，200 双倍；后台商品标价不变，实际结算按倍率扣款。普通餐食、拉面、矿泉水等日用品按日常消费处理，默认能把饱食度或水分大幅补回。休息基础恢复精力 55 点以上，在宿舍、医院、餐饮场所或明确付费使用设施时恢复更多。角色濒死会保留 1 点生命，但不扣除能力经验。

背包使用规则：玩家写“吃/喝/使用/用掉背包里的某物”“拿出随身便当/矿泉水”等，会优先消耗背包库存，不会扣学都币。背包里没有对应物品时只提示数量不足，不会自动购买；只有明确写“买/购买/点餐/付款/结账”等购买动作时才会花钱。

任务与成长规则：`任务/今日任务/引导` 会根据角色身份、阵营、当前位置和现实北京时间给出建议。学生在上课时段前往所属学校或能力开发课程可获得学业经验，夜间按时回宿舍或学校附近可避免声望损失；风纪委员/警备员可通过巡逻、失物、寻人、维持秩序提升声望；研究身份可做公开记录和设备维护；音乐、绘画、表演等会积累艺术经验。任务不是硬性按钮，只要在 `行动` 中自然写出对应行为即可触发。

NPC 与关系规则：NPC 会按现实时间、身份和地点活动，移动必须走地图路线。玩家找 NPC 时，如果 NPC 不在当前位置但能通过消息、支部记录或熟人转述联系，会给出有限反馈；不能合理联系时会提示对方主要活动地点。玩家/NPC、玩家/玩家之间的好感度只存在后台隐藏数值，玩家侧不显示具体分数，只通过“态度缓和、警惕、愿意配合”等剧情反馈体现。

路径事件规则：玩家一回合可按 `max_move_steps` 走多段地图边。前往远处地点时不必写出每个途经点，程序会自动找三格内路径；途经地点可能触发路边新闻、熟人擦肩、风纪委员/警备员介入或 NPC 按性格旁观/提醒。行动内容本身不限制做事数量，但地图移动距离仍受配置限制。

## 世界书落地内容

当前插件已按世界书默认生成：

- 23 个学区地图和路线连接
- 默认 NPC：第七学区日常角色、常盘台角色、风纪委员、警备员、医院、研究侧、暗部边缘、Level 5 传闻与幕后禁区角色
- 默认商店：水、拉面、便当、医疗、通勤、终端电池等日常物资
- 默认事件：大霸星祭、物流安检、地下街停电、能力测定日等公共事件
- 角色卡模板：身份、阵营、能力、能力等级、性别、穿衣着装、简易身材

## 多模型路由建议

先在 AstrBot 里把不同 API / 模型配置成不同 provider，再在插件配置里选择：

- `main_provider_id`：主模型，适合填 Claude 4.6o 一类综合能力强的模型。
- `story_provider_id`：剧情模型，当前每小时结算润色会优先走这里，适合填 DeepSeek v4 Pro。
- `fast_provider_id`：快速模型，用于自然语言角色卡整理，并预留给后续反作弊、摘要、轻量判定等任务，适合填 v4 Flash。
- `default_provider_id`：旧版兼容默认 provider，未配置专用模型时作为回退。

当前实际会产生大模型调用的是“每轮结算剧情润色”和“自然语言角色卡整理”。结算润色调用顺序是：`story_provider_id` -> `main_provider_id` -> 当前群会话 provider -> `default_provider_id`。自然语言角色卡整理调用顺序是：`fast_provider_id` -> `main_provider_id` -> 当前群会话 provider -> `default_provider_id`；API 失败会自动使用程序解析兜底。

## 大模型消耗说明

每轮结算最多调用一次大模型：程序把本轮行动、公共事件、NPC 动向、个人结果和可控长度的世界书节选打包成一个 JSON 请求，模型一次性返回公共摘要和各玩家私聊文本，插件再拆分私聊给已审核玩家。没有到结算周期不会持续调用；关闭大模型后不产生模型调用。

世界书默认节选 8000 字符，通常会额外增加几千 token 的输入消耗。若玩家多、每轮行动多，建议先把 `worldbook_prompt_chars` 调到 4000-8000 之间观察效果；如果只想使用程序模板，可关闭 `enable_llm`。

结算输出长度可在配置里调整：`public_summary_max_chars` 控制公共摘要最大字符数，`private_result_max_chars` 控制每个玩家个人结果最大字符数。调低可以减少输出消耗，调高可以让剧情更完整；默认分别是 1800 和 1400，当前上限均为 12000 字符。

长消息会按 `message_chunk_chars` 自动分段发送，默认每段 1800 字符，段间隔由 `message_chunk_delay_ms` 控制。

冲突不再按“过家家训练”处理：明确攻击、互殴、袭击、正面冲突会触发 PvP 伤害，默认生命损失区间由 `pvp_damage_min` / `pvp_damage_max` 控制；高风险行动额外受 `risk_damage_multiplier` 影响。

冲突强度可继续调节：`conflict_damage_multiplier_percent` 控制真实冲突伤害倍率，默认 130%；`conflict_narrative_intensity` 控制冲突文本强度，默认 3。插件不会尝试绕过或破解大模型安全限制，而是由程序硬结算伤害、濒死、警备介入和治疗压力，再要求模型做非露骨但更严肃的润色；模型输出过轻时会回退到程序结果。

如果玩家行动影响到其他玩家，例如殴打、袭击、强行互动等，即使目标玩家本轮没有提交行动，系统也会生成一条被动影响结果并尝试私聊发送给目标玩家，同时写入目标玩家状态变化。

如果玩家本轮没有行动，系统仍会按当前位置、时段和附近角色生成一条环境结果，例如宿舍隔壁动静、窗外脚步声、广播提醒、附近 NPC 的活动或公共事件余波，不会只发一条干巴巴的“你本轮没有行动”。

玩家交互会按类型处理：同行/组队、交流、支援保护、压力型互动、赠与/交易、冲突。系统会为双方补充私聊反馈，小幅调整玩家之间的关系分；涉及明确金额或背包物品名称时，会尝试完成学都币或物品转移。

资源转移采用“支出方确认”规则：只有行动者写明自己给出/转账/赠送自己的学都币或背包物品，才会过账；单方面声明“某玩家给我钱/物品”会被反作弊拦截；“抢夺/偷取/勒索”会按冲突或压力行动结算，但不会直接把对方资源转到自己名下。
